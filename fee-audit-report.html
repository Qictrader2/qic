Vendor Fee (Seller's Escrow Fee)

// Ceiling division — platform never under-collects
fn vendor_fee(amount_minor: i64, bps: u32) -> i64 {
    let product = amount_minor as i128 * bps as i128;
    ((product + 9_999) / 10_000) as i64
}

Reseller Commission Split

// Reseller's gross (floor — never over-receives)
fn reseller_gross_commission(
    amount_minor: i64, markup_pct: f64
) -> i64 {
    (amount_minor as f64 * markup_pct / 100.0)
        .floor() as i64
}

// Platform's cut of reseller commission (ceiling)
fn reseller_fee(
    gross_commission: i64, reseller_fee_bps: u32
) -> i64 {
    let product = gross_commission as i128
                * reseller_fee_bps as i128;
    ((product + 9_999) / 10_000) as i64
}

⏱ Trade Lifecycle Timeline — Complete Audit Trail

Trade Created
Quote calculated via MPR-005. Fiat fee deducted from buyer input. Crypto amount and escrow fee computed.
trades.fee_amount = fiat_fee (f64) · escrows.fee_amount = crypto_fee (i64)
Escrow Funded
Seller's balance locked. Escrow created with fee_amount embedded. Atomic transaction (TF-001).
LEDGER → escrow_lock (debit, seller, full amount including fee)
Buyer Confirms Payment
Trade transitions to Paid status. Waiting for seller confirmation.
trade.status = Paid
Trade Completed — Settlement
Custodial: Atomic 3-way split — buyer credited, reseller credited (if resell), platform credited.
On-chain: Blockchain tx to buyer address + fee address. If tx fails → GAP-006 orphaned fee recorded.
LEDGER → trade_debit (vendor) · escrow_release (buyer) · fee (platform) · fee (reseller, if applicable)
Affiliate Commissions Recorded
Referral chain resolved (buyer → L1 → L2 → L3). Commission calculated per level from escrow fee. Earnings inserted, profiles updated, tier promotion checked.
LEDGER → affiliate_commission (credit, per referrer per level) · affiliate_earnings INSERT · affiliate_profiles UPDATE
Tier Promotion Check
If referrer meets both volume AND affiliate count thresholds, tier is upgraded. One-way progression — no demotion.
affiliate_profiles.tier = new_tier (if conditions met)

High Risk
1
Quote vs. escrow fee mismatch
HIGH Quote Fee vs. Escrow Fee Mismatch
Quote uses fiat fee: fee = fiat_amount × bps / 10,000 (ZAR, f64).
Escrow uses crypto fee: fee = crypto_amount × bps / 10,000 (minor units, i64).
These may differ due to exchange rate rounding. The trade.fee_amount (f64) may not match what was actually locked in escrow (i64). This means the fiat fee snapshot shown to the user at quote time can diverge from the actual crypto fee deducted.
MEDIUM Affiliate Earnings Not Fully Atomic
affiliate_earnings INSERT and affiliate_profiles UPDATE are separate queries. If the UPDATE fails (e.g., no matching affiliate_profiles row), the earning is recorded but aggregate stats (pending_earnings, total_earnings) are not updated. Mitigated by checking for affiliate_profiles.id before inserting, but not transactionally guaranteed.
MEDIUM No Fee Address Audit
The platform fee address comes from config.fee_address_for_network(&network). No immutable record is stored of which address actually received the fee at settlement time. Only the tx_hash in the ledger entry links the fee to the blockchain — if the config changes, historical reconciliation requires cross-referencing on-chain data.
MEDIUM Orphaned Fee Recovery is Manual
GAP-006 orphaned_fee_transactions with status 'pending' can sit indefinitely. resolved_by and resolution_note are optional — no enforced escalation workflow. Risk: unclaimed on-chain fees could remain unrecovered without alerting anyone.
LOW Affiliate Tier Promotion Async
check_and_promote is called after record_commissions but if commission recording fails, the tier check may not execute. Since tiers only progress upward and the check runs on every subsequent trade, a missed promotion is self-correcting on the next successful trade.

Anti-Patterns to Avoid

Sankey Diagram
Accessibility grade C, breaks on mobile, needs d3-sankey (not installed), and you'd build the fallback table anyway.
Pie / Donut Charts
Grade C for colorblind users. QIC has up to 5 fee parties — exceeds recommended slice count.
Color-Only Differentiation
Always pair color with text labels or icons. Current getLevelColor() is correct — don't regress.
Hardcoded Hex in JSX
Use var(--brand-blue) etc. Existing EarningsChart hardcodes #009CFF — new components should use CSS variables.
